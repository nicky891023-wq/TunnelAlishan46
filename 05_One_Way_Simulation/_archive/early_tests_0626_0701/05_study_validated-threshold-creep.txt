### THE THRESHOLD FISH: COMPLETE VALIDATED CODE

Event 23 = **SOLVED callback** (fires after each model step/cycle, used to apply custom logic each solve event).

#### Full Threshold Definition (WadeCreepSupport.f3dat verbatim):
```fish
[threhold = 0.6]  ; Tunable reduction factor (0.4, 0.5, 0.6 tested)

fish define threshold
    loop foreach z zone.list
        if zone.group(z) = "rock"
            ; Effective stress (subtracting pore pressure)
            local sig1 = -zone.stress.prin.x(z)
            local sig3 = -zone.stress.prin.z(z)
            local pore = zone.pp(z)
            local pmean = (sig1 + sig3)/2 - pore
            
            ; Deviatoric stress (half the difference of principal stresses)
            local q = (sig1 - sig3)/2
            
            ; Zone material properties
            local c = zone.prop(z, 'cohesion')
            local phi = zone.prop(z, 'friction')
            
            ; Reduced failure envelope (threshold-scaled)
            local ceff   = threhold * c
            local phieff = threhold * math.tan(phi * math.degrad)
            local q_threshold = ceff + pmean * phieff
            
            ; Creep activation: if q >= q_threshold, creep ON
            if q >= q_threshold
                zone.prop(z, 'viscosity-kelvin')  = [etak]
                zone.prop(z, 'viscosity-maxwell') = [etam]
                zone.group(z) = "CreepActive"
            else
                ; Creep OFF: set viscosities to very high (1e100)
                zone.prop(z, 'viscosity-kelvin')  = 1e100
                zone.prop(z, 'viscosity-maxwell') = 1e100
                zone.group(z) = "CreepOff"
            endif
        endif
    endloop
end

; Register callback at event 23 (each solved cycle)
fish callback add @threshold 23
```

**Physical interpretation**: q = deviatoric stress (shear intensity); q_threshold = Mohr-Coulomb envelope scaled by 0.6×. When stress-state (q, pmean) crosses the threshold, viscosities drop from 1e100 (frozen) to etak/etam (active), enabling creep deformation. Water rise lowers pmean (via increased pore pressure), shrinking the threshold, thus triggering creep.

#### Per-Stage Structure (Threshold02/04/06.f3dat + Chapter4 Creep.f3dat):

**Stage initialization**:

**Multi-stage creep loop (Threshold02 pattern)**:
```fish
[real_totaltime = 3600*24*1500]    ; Total days → seconds (1500 days)
[time_step = 300]                  ; Per-step increment (seconds)
[check_segment = 5]                ; Save at 5 checkpoints

fish define creep(ts, tts, cs)
    loop n (1, cs)
        savename = string(n)
        command
            model creep timestep fix [ts]
            model solve creep time-total [tts*n]
            model save @savename
            zone gridpoint initialize velocity (0,0,0)  ; ZERO velocity each stage
        endcommand
    endloop
end

@creep([time_step], [real_totaltime/check_segment], [check_segment])
model save 'final'
```

**Per-stage time formula**: 
- `model solve creep time-total [X]` = solve TO absolute cumulative time X (not increment). 
- E.g., stage 1 solves to 300 days, stage 2 to 600 days, stage 3 to 900 days, etc.

#### Velocity Zeroing Pattern (after each stage):
```
model solve creep time-total [tts*n]
zone gridpoint initialize velocity (0,0,0)  ; CRITICAL: reset velocity after each stage
model save @savename
```
**Rationale**: Prevents velocity "ratcheting" between stages; new stage starts from zero velocity baseline.

- **0-30d (low water)**: Almost no creep → `disp-time ≈ flat` (stress well below threshold).

### GOTCHAS & VALIDATION NOTES

1. **Event 23 = "solved" callback**: Fires every cycle/step after mechanical/creep solve. Wade relies on this to re-evaluate threshold each step.

2. **Pore pressure → effective stress**: Always use `pmean = (sig1 + sig3)/2 - pore` (NOT zone.stress.mean). The threshold checks *effective* stress.

3. **Threshold is tunable**: Wade tested 0.2, 0.4, 0.5, 0.6. Higher = stricter (creep activates later); lower = more permissive. **0.6 is Wade's validated default**.

4. **Burgers-mohr property order** (critical):
   ```
   zone cmodel burgers-mohr
   zone property bulk ... shear-maxwell [Gm]
   zone property shear-kelvin [Gk] viscosity-kelvin [etak] viscosity-maxwell [etam]
   ```
   **Both bulk and shear-maxwell must be set together**; then shear-kelvin + viscosities.

5. **Water level changes do NOT auto-equilibrate pore pressure**: You must call `model solve fluid ...` explicitly after changing `zone face apply pore-pressure`.

6. **Displacement/velocity zeroing between stages**: Wade uses `zone gridpoint initialize velocity (0,0,0)` after every stage. Displacement zeroing is optional (used if tracking incremental per-stage, not cumulative).

7. **Coupled model wall-zone facet degeneration**: Gradual loading (many small excavation steps) prevents facet collapse. One-shot excavation + creep can degenerate facets → **use staged excavation in Excavation.f3dat** before Creep.f3dat.

8. **Shell support feedback**: Wade's `@shell_failure` and `@shell_stress` FISHs (callback event 12) manage support stiffness degradation under high stress. These are *in addition to* the threshold creep callback (event 23).
