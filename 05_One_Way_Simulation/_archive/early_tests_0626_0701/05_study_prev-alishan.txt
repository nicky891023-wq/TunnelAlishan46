### **KEY LESSONS / WHAT WENT WRONG / TO AVOID**

**[L1] FISH Parser UTF-8 Bug** (RECALIBRATION_LOG §5):

- Using zone-list position indices in `insitu_f` map under multi-core callback → wrong zones matched.
- **FIX:** Use zone-keyed map: `insitu_f(zone.id(z))` — O(1) lookup, no ordering risk.

**[L3] η_M Too Small → Unbounded Boundary Creep** (RECALIBRATION_LOG §3, RECALIBRATION_LOG §6):
- 6.0e15 produced 1.4 m/yr boundary displacement — physically unrealistic (~100× field rate).
- **FIX:** Increase to 4.0e17 (τ_M = η_M / G_M ≈ 25 yr secondary relaxation time).

**[L4] Plastic Collapse Before Creep (RECALIBRATION_LOG §5):**
- Large model initial equilibrium with `solve elastic only` + user-set cohesion → zone residual strength not applied → 1 m displacement from plastic failure, NOT creep.
- **DIAGNOSIS:** diag_yield.f3dat confirmed 100% yield in layer1, 99.9% in layer2 → FoS ≪ 1.
- **FIX:** Use bare `model solve elastic` (no "only" flag); then separately raise c/φ if needed. Layer2/3/4 now have c ~×5–10 higher (per RECALIBRATION_LOG table §3).

**[L5] In-Situ Creep Reference Must Survive Restore** (crp_process_v3.f3dat lines 68–104):
- FISH maps are lost on `model restore`. Small model must write `insitu_f` to file after stage00, then reload after each subsequent `model restore`.
- **FIX:** Use `write_insitu_ref(fname)` + `read_insitu_ref(fname)` pair; store in stage-persistent file.

**[L6] Zone-Ring v3 Geometry** (RECALIBRATION_LOG §9):

**[L7] Wet-Stage Convergence Risk** (ANALYSIS_PIPELINE §2.3):
