**Rigid-Body Removal**

Keep subtracting best-fit translation + rotation for production. The code intentionally drives the deformational residual in [couple_idw.py](C:/Users/Wade/Desktop/Tunnel_TX/05_One_Way_Simulation/couple_idw.py:53). Switching to translation-only or no subtraction is a sensitivity/upper-bound case, not the corrected one-way BC, unless you also move every rigid PFC wall consistently with the removed rigid mode.

**Free-`w_inner` Implementation**

Delete `w_inner`; do not make it servo-free. PFC walls are kinematic boundaries; a servo is an active controller, not a passive free surface. Official FLAC3D/PFC examples use `wall delete wall range name 'top'`, so the corresponding command is:

```text
wall delete wall range name 'w_inner'
```

Numerical stability: the bonded lining balls should not “fly” if you do a release-only baseline with damping/calm and velocity/crack breakers. If deleting `w_inner` alone causes major cracks, that is a result: the rigid inner wall was carrying artificial support, and the free-inner run must be reported relative to the free-inner settled baseline.
