### **LARGE → SMALL COUPLING (ApplyVel_IDW)**

**Mechanism:**  
The large model's displacement field is **transferred to the small model as a VELOCITY boundary condition**, applied to the small model's outer gridpoints over the 30-day creep stage.

**Files:**
- **Export**: C:/Users/Wade/Desktop/ClassPhD/test/TX/260529/02_small_model/helpers/io_disp.f3dat (lines 3–63: `Export_Local_Disp`)
- **Mapping**: C:/Users/Wade/Desktop/ClassPhD/test/TX/260529/02_small_model/helpers/io_disp.f3dat (lines 452–594: `ApplyVel_IDW`)
- **Invocation**: C:/Users/Wade/Desktop/ClassPhD/test/TX/260529/02_small_model/helpers/crp_process_v3.f3dat (line 366: `[ApplyVel_IDW(t_period, 100)]`)

**Process:**

1. **Large model export** (after each of the 12 stages):
   ```
   Export_Local_Disp(file_name)
   ```
   Extracts gridpoints in the small model's footprint (±100 m around the small box) and writes cumulative displacement (`ux, uy, uz`) to a `.dat` file in format:
   ```
   gpid x y z ux uy uz
   ```

2. **Read into memory**:
   ```
   ReadFile_disp(file_name)
   ```
   Loads displacement data into 6 global arrays: `xdat, ydat, zdat, uxdat, uydat, uzdat`.

3. **IDW interpolation + velocity conversion** (`ApplyVel_IDW`):
   - For each small model boundary gridpoint:
     - **IDW search**: finds all large model data points within `search_radius` (typically 100 m)
     - **Weight formula**: `w = 1 / d²` (inverse distance squared)
     - **Interpolated target displacement**: `target_u = Σ(u_i × w_i) / Σ(w_i)`
     - **Fallback**: if no points in radius, uses nearest neighbor
   
   - **Velocity = (target − current) / creep_time**:
     ```FISH
     gp.vel.x(igp) = (target_ux - curr_ux) / creep_time
     gp.vel.y(igp) = (target_uy - curr_uy) / creep_time
     gp.vel.z(igp) = (target_uz - curr_uz) / creep_time
     ```
     (Lines 573–575 of io_disp.f3dat)

   - **Fixity**:
     - **Boundary gridpoints**: `gp.fix(igp) = vector(1,1,1)` → velocities FIXED → drive the model
     - **Internal gridpoints**: `gp.fix(igp) = vector(0,0,0)` → FREE to react internally (tunnel, creep)
   
   This is the **"FullMapping" philosophy** (lines 343–445) adapted to IDW.

**Displacement → Velocity Conversion:**
- `creep_time` is the stage duration in **seconds**: `t_period * 24*60*60` (line 307, crp_process_v3.f3dat)
- For 30-day stages: `creep_time = 30 × 86400 = 2,592,000 sec`
- For 5-day intermediate stages: `creep_time = 5 × 86400 = 432,000 sec`
- Velocity is thus the **incremental displacement** divided by the time span → naturally becomes the boundary rate over that stage

**Zeroing/Initialization:**
- **NO explicit zeroing** of displacement at stage hand-off
- Each stage reads the cumulative displacement from the large model (starting from day0)
- The small model gridpoints accumulate their own displacement independently
- At each stage, the velocity is recomputed as `(target_cumulative - current_cumulative) / new_dt`
- **Reference point for relative disp**: Calc_Relative_Disp(1300, 900, 1740) anchors to a fixed interior point; all boundary convergence is reported relative to that
